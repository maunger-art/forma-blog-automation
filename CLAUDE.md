## Mission: Phase 1A of the SEO & Authority Engine (Days 1–15)

Reference doc: `Forma_SEO_Engineering_Implementation_Plan.docx` (with Mike). The full plan is layered, but Phase 1A in this repo is:

1. **`shared/taxonomy.json`** — canonical cluster definitions. Five initial clusters: `zone-2-training`, `hrv-readiness`, `training-load`, `garmin-wearable`, `marathon-training`. Each entry has: `id`, `name`, `pillar_slug`, `pillar_url`, `related_tool`, `seed_terms[]`, `classification_keywords[]`, `supporting_article_slots`, `programmatic_page_quota`.

2. **`shared/content_metadata.json`** — generated by `forma_blog_build.py` at build time from `posts_manifest.json`. One entry per published article: slug, title, date, cluster, URL.

3. **`linking/link_map_builder.py`** — reads `posts_manifest.json` + `seo/seo_pages_manifest.json` + `shared/tool_registry.json`. For each piece of content, computes up to 5 internal links: same-cluster pillar (priority 1), 2 same-cluster supporting articles, 1 cluster tool, 1 product feature page. Output: `shared/link_map.json`.

4. **`linking/link_injector.py`** — invoked from `forma_blog_build.py` after HTML is rendered. Uses BeautifulSoup. Caps at 5 injected links per page. Skips text already inside `<a>`. Wraps in try/except — if parsing fails, returns original HTML unchanged.

5. **Schema upgrades** in `forma_blog_build.py`:
   - `BreadcrumbList` JSON-LD on every article and the index
   - `FAQPage` JSON-LD on any post with an `faq` field in its manifest entry
   - Keep existing `Article` / `BlogPosting` schema

6. **`seo/seo_pages_manifest.json`** — five programmatic Q&A page entries to start. Schema: `slug`, `question`, `cluster`, `template`, `answer_summary`, `body_html`, `forma_insight`, `faq[]`, `cta_tool`, `cta_feature`, `meta_description`, `published`, `last_updated`.

7. **`seo/seo_page_builder.py`** — generates one HTML page per published manifest entry into `output/seo/`. Reuses `BRAND_CSS`, `NAV`, `FOOTER`, `DRAWER_JS` from the article builder for visual consistency. Inject `FAQPage` + `BreadcrumbList` schema. Wire into the existing build workflow so SEO pages deploy alongside articles.

## Working principles

- **Source of truth is the manifest, never the rendered HTML.** Don't edit files in `output/` — change manifests/templates/builders and rebuild.
- **Feature flags for risky additions.** New behavior should be toggleable via a constant at the top of the relevant module (e.g., `LINK_INJECTION_ENABLED = True`).
- **Don't fail the build on optional pipelines.** Question harvest, GSC import, and link map updates should commit on their own and never block the article build.
- **Cross-repo contract is JSON only.** Never import from the React repo or vice versa.
- **Terminal commits.** When work is ready: `git add … && git commit -m "…" && git push`. Cloudflare/GitHub Pages auto-deploys.
